KEY INSIGHTS SYNTHESIZED
================================================================================

1. TILE TOPOLOGY IS NOT OPTIONAL
   Sapphire Rapids' tile design fundamentally changes inference architecture.
   NUMA-aware scheduling (SNC=4) is prerequisite for 2-5x latency improvement.

2. LATENCY VARIANCE BEATS THROUGHPUT
   For inference SLA compliance, 92.5% jitter reduction is more valuable than
   10% throughput improvement. This inverts typical optimization priorities.

3. FIXED FREQUENCY > DYNAMIC TURBO
   For tail latency (p99), disable turbo boost in favor of conservative fixed
   frequency. Frequency throttling jitter dominates latency percentiles.

4. NUMA IS PERVASIVE
   Modern inference workloads are inherently NUMA-sensitive. Ignoring NUMA
   gives away 20-40% performance automatically.

5. COMPILER LIMITATIONS
   Modern compilers cannot generate optimal DSB, NT store, and CLDEMOTE patterns.
   Manual optimization is necessary for last 20% of performance.

6. POWER MANAGEMENT MATTERS
   RAPL power limiting and frequency scaling are not "nice to have" but core
   part of achieving consistent latency under load.

7. CORE ISOLATION IS GAME-CHANGING
   nohz_full isolation eliminates 50+ µs latency spikes caused by interrupts.
   The 10% throughput loss is more than compensated by latency SLA achievement.

8. TILE AFFINITY DRIVES EVERYTHING
   For batched inference, pinning batch elements to tile boundaries (12 cores
   at a time) is single highest-impact optimization across all modules.

  ☐ Set RAPL power limit to 280-300W
  ☐ Disable C-states (frequency scaling)
  ☐ Enable Xeon Max features if using HBM variant

  ☐ Boot with nohz_full=12-47 (reserve cores)
  ☐ Boot with rcu_nocbs=12-47 (RCU isolation)

  ☐ Disable transparent huge pages (echo never > /sys/.../transparent_hugepage/enabled)
