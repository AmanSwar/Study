Key Insight: Your algorithm's complexity doesn't matter if you miss the cache.
An O(n²) algorithm with perfect cache locality runs faster than O(n log n) with
poor locality. Modern workloads are memory-bound, not CPU-bound.

buffers. Understand which memory order (relaxed vs acquire/release vs seq_cst)

MODULE 4: OPERATING SYSTEM INTERNALS FOR PERFORMANCE ENGINEERS
================================================================================
Lines: 1,187
Topics:
  • Kernel vs user space: syscall boundary, cost (~500 ns overhead)
  • vDSO (Virtual DSO): syscall avoidance (5x faster than syscalls)
  • Process scheduling: CFS, CPU affinity, isolcpus
  • Interrupt handling and IRQ affinity (preventing cache destruction)
  • Memory allocation internals: ptmalloc2, jemalloc, tcmalloc
  • Huge pages via mmap(MAP_HUGETLB), transparent huge pages
  • cgroups and resource isolation (CPU limits, memory limits)
  • perf_event_open: hardware counter access

Key Insight: The kernel's default fair scheduler destroys performance. Use CPU
affinity (free 5-20% speedup) and interrupt affinity to prevent context switches
and cache misses. Custom memory pools beat malloc for inference engines.

PERFORMANCE RELEVANCE FOR ML SYSTEMS ENGINEERS
================================================================================

Why These Modules Matter:

Module 1 (Execution Model):
  → Understand call stack overhead in token-by-token inference loops
  → Eliminate unnecessary function call prologue/epilogue
  → Know why UB enables aggressive optimizations that break code

Module 2 (Memory Hierarchy):
  → Model weights are 10-100 GB: TLB and cache behavior critical
  → Attention kernel is O(seq_len²): memory bandwidth is bottleneck
  → Pre-load KV cache for inference to maximize cache hit rates

Module 3 (Memory Consistency):
  → Inference pipeline has multiple threads (batch processor, token producer)
  → Lock-free queues for job distribution (no blocking)
  → Understand memory ordering for safe concurrent access to KV cache

Module 4 (OS Internals):
  → Latency-critical inference requires CPU affinity and isolated cores
  → Custom memory pools replace malloc (zero allocation overhead)
  → Interrupt affinity prevents cache misses from network I/O

Real-World Speedups Demonstrated:
  • False sharing fix:        15.3x
  • Prefetching:              29-53% faster
  • CPU affinity:             5-20% speedup
  • Huge pages (TLB):         10-30% for large models
  • Lock-free (vs mutex):     10x for high contention
  • Vectorization (restrict): 4-8x for element-wise ops
  • Cache-oblivious matmul:   3-5x vs naive implementation
